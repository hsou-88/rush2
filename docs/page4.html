QA

要件理解の為に疑問を書き出します

英語版pdfの読解

QA形式で壁打ちしながら整理します。

事例

  • The dictionary will have the following rules:[a number][0 to n spaces]:[0 to n spaces][any printable characters]\n

辞書の形式は数値の後にスペースとコロンが続いて，さらにスペースが含まれた後に任意の印刷可能な文字が続く形になる．

  • The dictionary will always have at least the keys contained in the reference dictionary. Their values can be modified, more entries can be added, but the initial keys can’t be removed.

配布された辞書に入っていたキーは必ず含まれていることが保証されている．単語に関しては変更可能で，更なるキーと単語の追加は可能．

  • The entries of the dictionary can be stored in any order.

辞書はソートされておかれている必要はない．行の入れ替えに関しては自由に置かれている固定のループ処理をしていると条件分岐でエラーがおきる

  • 値が変更されたらどのようになるのが適切か

位のルールを維持したまま，辞書に登録された文字を出力するのが正しいそう．

  • Your program should handle numbers beyond the range of unsigned int

unsigned int は0 ~ 4,294,967,295までの値に対応.

辞書に1000000000000000000000000000000000000: undecillionがあることからlonglong int と atoiを使って数値処理する実装は今回使えなさそう

  • 文字列で扱うということは，言語の構文解析のようなものに近いだろうか?

似たような事例を見つけることはできた．CSVファイルのパース問題で，CSVはカンマ区切りに成型されていることからそれがコロンに置き換わり，単語と数値につながりができたイメージ

一番簡単な問題がCSVファイルの解析でそこから考えていけば，難易度を緩やかに上げられそう．

例えばこれ
  • :(コロン)を外したり，keyとvalueの入れ替えは今回可能なのか

できない，実装で考えてみると，valueに数字が入る可能性もあり，コロンがなければ解析ができない．同様にkeyとvalueが入れ替わるとvalueに数字が入る可能性もあるので，keyとvalueの明確な区別ができなくなる.

  • パースエラーとして扱うような例

コロン : が複数ある，左側が数値でないとき,左側が空のとき,右側が空,スペースのみ含まれるとき(スペースはtrimされるから右側が空と同じ扱いになる)

  • If the dictionary does not allow you to perform the conversion of the provided number, your program must output "Dict Error\n".

入力の数値に対して，対応する文字列がないときはDict Errorとする一部が存在するからと言って翻訳はしない.

  • ハッシュ,ソート済み配列,構造体ノードどれがいいか

構造体がいいように思えているが，フローチャートなどで実装のイメージを作ってから判断

  • If you have any errors from the dictionary parsing, your program must output "Dict Error\n".

一つでも辞書形式の不正があったら，エラー．変換できる値が来ていたとしてもだめ．
